---# 🖼️ 小红书图片下载指南

> 适用场景：从小红书分享链接下载图片/壁纸
> 最后更新：2026-06-20
> 关联指南：`小红书视频下载指南.md`

---

## 📋 前置条件

| 项目 | 说明 |
|:---|:---|
| 核心思路 | 从网页版HTML源码中提取 `og:image` 标签获取高清图片直链 |
| 推荐工具 | `extended_http_tools`（扒源码）+ `super_admin:shell`（curl 下载）|
| 可选工具 | 浏览器工具（`browser:evaluate` 获取元素链接）|

---

## 🚀 操作流程（三步走）

### Step 1️⃣ 获取网页源码

用桌面版 UA 访问小红书分享链接：

```
extended_http_tools:http_request
├── url = 小红书分享链接（xhslink 短链）
├── method = GET
└── headers = { "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 ..." }
```

> ⚠️ **关键**：必须用桌面版 Chrome UA！移动版 UA 返回的页面可能不包含完整内容。
> ⚠️ 响应内容较大时会自动保存到文件，然后用 `grep_code` 检索。

---

### Step 2️⃣ 提取图片直链

在 HTML 中搜索 `og:image` 标签：

```
grep_code
├── path = 上一步保存的 HTML 文件路径
└── pattern = og:image
```

找到形如以下的行，复制 `content` 属性值：

```html
<meta name="og:image" content="http://sns-webpic-qc.xhscdn.com/.../...!nd_dft_wlteh_jpg_3">
```

> 💡 **小贴士**：
> - 图文笔记通常有多个 `og:image`，按顺序对应第1张、第2张……
> - 图片链接后缀 `!nd_dft_wlteh_jpg_3` 表示「网页版预览尺寸」，实际已经是高清（1080px+）
> - 如果想获取原图，可以尝试去掉 `!nd_dft_...` 后缀部分，但通常预览版已够用

---

### Step 3️⃣ 下载图片

#### 方法 A：curl 下载（推荐，可绕过防盗链）

使用 `super_admin:shell` 的 `curl` 命令，**必须带 Referer 头**：

```
super_admin:shell
└── command = curl -L \
    -o "/storage/emulated/0/Pictures/美图/壁纸01.jpg" \
    -H "Referer: https://www.xiaohongshu.com" \
    -H "User-Agent: Mozilla/5.0 ..." \
    "https://sns-webpic-qc.xhscdn.com/.../...!nd_dft_wlteh_jpg_3"
```

> ⚠️ **重要**：不加 `Referer: https://www.xiaohongshu.com` 会返回 403 防盗链错误！
> ✅ `exitCode: 0` 即为成功，输出为空是正常现象。

#### 方法 B：下载工具 `download_file`（备选）

```
download_file
├── url = 图片直链
├── destination = 保存路径
└── headers = { "Referer": "https://www.xiaohongshu.com", "User-Agent": "Mozilla/5.0 ..." }
```

> ⚠️ 注意：小红书对 `download_file` 的多部分下载有限制，可能失败。**方法 A 更稳定**。

#### 方法 C：浏览器工具辅助（图片已加载在页面中时）

```
browser:evaluate
└── function = () => { /* 获取所有 img 并提取 xhscdn 链接 */ }
```

然后结合 `super_admin:shell` 的 curl 下载。

## 📌 常见问题 & 避坑指南

### ❌ 下载失败 / 403 Forbidden
- **原因**：缺少 Referer 或 User-Agent 不对
- **解决**：请求头必须包含 `Referer: https://www.xiaohongshu.com`
- **终极方案**：使用 `super_admin:shell` 执行 `curl -H "Referer: ..."`，几乎不会失败

### ❌ 图片链接找不到
- **原因**：短链跳转后可能不是标准笔记页，或页面需要登录
- **解决**：先用 `visit_web` 访问短链观察是否正常显示，再提取最终 URL 进行 HTTP 请求

### ❌ 下载的文件是 webp 格式，相册打不开
- **原因**：小红书 CDN 默认输出 webp 格式
- **解决**：webp 是通用格式，大多数现代相册都支持。如果确实需要 jpg，可用 FFmpeg 转换：
  ```
  ffmpeg:ffmpeg_execute
  └── command = -i 输入.webp 输出.jpg -y
  ```

### ❌ 怎么判断是图文笔记还是视频笔记？
- 图文笔记：`<meta name="og:type" content="article">`，有多个 `og:image`
- 视频笔记：`<meta name="og:type" content="video">`，有 `og:video` 标签
- 可参考同目录下的 `小红书视频下载指南.md` 处理视频

## 🎯 完整操作示例（实战模板）

```
1️⃣ 获取 HTML 源码
   http_request → GET 分享链接（桌面 UA）

2️⃣ 搜索图片链接
   grep_code → pattern="og:image" → 得到 jpg/webp 直链

3️⃣ 下载图片
   super_admin:shell → curl -L -o 保存路径 \
     -H "Referer: https://www.xiaohongshu.com" 图片直链

4️⃣ （可选）webp 转 jpg
   ffmpeg_execute → -i 图片.webp 图片.jpg -y
```
