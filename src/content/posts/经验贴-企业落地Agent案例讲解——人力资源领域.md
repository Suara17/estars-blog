## 背景

在公司年终复盘时，人力考核指标涉及离职率。公司近一年离职员工较多，离职率飙升。人力部门认为业务部门没有带好新员工，业务部门则认为人力招来的人不符合需求，双方僵持不下。同时，人力部门时间和精力有限，并非每一位 HR 都拥有丰富经验。为了帮助人力部门提效并找到更符合公司定位的人才，开发了一套 **多 Agent 合作的人力招聘系统**。

---

## 项目构成：四 Agent 协同

### 1. 打招呼 Agent
- **功能**：调用 RPA 每日自动上 Boss 直聘抓取在线简历并分析，判断是否符合岗位要求。符合则自动打招呼，并定时监控是否有回复。
- **处理方式**：有回复时将对话截图存到人力 Web 端口。
- **AI 角色**：定时调度影刀 RPA + 识别简历匹配度。
- **输出**：形成在线表格，可查看聊天记录、在线简历、匹配点，并出具评分和匹配度；求职者发送其他信息时，通过飞书/企微/钉钉机器人推送待查看消息。

### 2. 简历分析 Agent
- **简历来源**：
  1. Boss 直聘平台获取
  2. 公司投递邮箱获取
  3. Web 端上传按钮
- **功能**：分析简历并形成简历库，支持按岗位筛选，支持同岗位候选人 **PK 对比**，输出优劣势意见给 HR。
- **⚠️ 注意**：不得有职业歧视（性别、年龄等需脱敏），必须针对工作经历分析。

### 3. 面试题 Agent
- **第一步**：业务部门提供重点关注维度、入职后需解决的问题、个性化关注点，打包为 Skill。AI 结合这些输出面试题，供 HR 和业务部门参考，可反复修改完善。
- **第二步**：经求职者同意签订知情同意书后进行 **飞书妙记** 录制，AI 将面试回答进行评分，提醒面试官二面/三面重点关注的点，或结合上一次面试内容出具新面试题。

### 4. 绩效考勤通 Agent
- **功能**：接入 HR 系统中的考勤和绩效考核数据。
- **转正答辩参考**：通过则记录优点，不通过则记录被淘汰的点。
- **数据敏感性**：AI 只记录岗位下曾出现过的问题（如经常迟到、项目未完成等），禁止出现「不顺眼」「价值观不符合」等主观理由。
- **额外价值**：也是对管理层的考核，避免随意开除员工。

---

## 项目细节

### 前置准备
- 招聘岗位的详细 JD（岗位说明书 + 所做的事情）
- 历史所有入职转正/未转正简历及理由
- 入职后绩效考核表与考核内容
- **数据安全与权限**：
  - 数据脱敏（身份证号等全部脱敏）
  - 仅开放给 HR 和业务 Boss 查看
  - 内网登录且记录每次操作日志
  - AI 编程软件为企业版，已签合约并出具体方案保证数据安全

### 开发周期
- 整体开发耗时 **3 个月**
- RPA 是绕不开的一环（影刀 RPA），目前仅对接 Boss 直聘平台

---

## 运行效果与反馈

- **已运行**：3 个月，整体流程已跑通
- **初期阵痛**：HR 工作量未减少，甚至有所增加，业务也一度反感
- **Boss 支持**：认为项目是长期价值
- **数据成果**：新入职员工与领导沟通更和谐，过去「对人不对事」的问题改善为「对事不对人」
- **留存率**：新员工留存率提高，大部分通过考核
- **3 个月后**：业务 Boss 几乎每天登录人力系统查看点击率上升
