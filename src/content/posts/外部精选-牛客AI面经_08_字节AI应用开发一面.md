# 🎯 字节 AI 应用开发一面 面经

> **作者**：找工作小鲸  
> **发布**：2025-05-14  
> **来源**：牛客网（搜索结果页摘要，全文因登录限制不可访问）  
> **备注**：14 道题，技术深度较高，涉及强化学习在 Agent 中的应用

## 📋 面试题目

### 基础与框架

1. 自我介绍
2. 你如何理解 **AI Agent**？它和传统 AI 应用的区别在哪里？
3. 用过哪些 Agent 框架（LangChain、CrewAI、AutoGen 等）？它们的优缺点？

### 场景题：客服场景的幻觉治理

4. **幻觉问题**：在客服场景中，大模型经常会产生幻觉，你是怎么治理的？有哪些有效的策略？
5. **知识注入**：如果客服需要接入新的产品知识（新的业务规则），你是直接更新 Prompt、做 RAG、还是微调？各有什么优劣？
6. **泛化性损失**：引入新知识后，发现模型在其他问题上的回答反而变差了，你怎么分析和解决这个泛化性损失问题？

### Expert Agent 架构

7. **Agent 划分维度**：你设计 Expert Agent（专家智能体）时，是按什么维度来划分的？按领域/按功能/按用户角色？
8. **SFT 训练数据构造**：如果你要为 Expert Agent 构造 SFT 训练数据，你会采用什么策略？数据的质量和数量哪个更重要？
9. **SFT vs RLHF**：什么时候用 SFT，什么时候用 RLHF？它们的适用范围分别是什么？

### 强化学习与对齐

10. **PPO 在 Agent 中的应用**：PPO（Proximal Policy Optimization）的奖励函数是怎么设计的？在 Agent 场景下有什么特殊的挑战？
11. **奖励模型训练**：如果 Expert Agent 的某个回答用户不满意，奖励模型应该如何调整？

### 多智能体协作（MAS）

12. **MAS 协作机制**：多个智能体之间如何通信和协调？你了解哪些主流的多智能体协作协议？
13. **局部优化问题**：在 MAS 中，每个 Agent 都在优化自己的目标，可能会导致整体次优。你遇到过吗？怎么解决？
14. **评估体系**：多智能体系统的整体效果如何评估？单 Agent 指标和系统级指标分别应该关注什么？

## 📊 知识点图谱

| 领域 | 知识点 | 重要度 |
|:---:|---|:---:|
| 幻觉治理 | 检索增强 / 提示约束 / 后处理校验 | ⭐⭐⭐⭐⭐ |
| Expert Agent | 划分维度 / SFT 数据 / 模型对齐 | ⭐⭐⭐⭐⭐ |
| 强化学习 | PPO 奖励函数 / RLHF / 奖励模型 | ⭐⭐⭐⭐ |
| MAS 协作 | 通信协议 / 局部优化 / 评估体系 | ⭐⭐⭐⭐ |
| 框架选型 | LangChain / CrewAI / AutoGen 对比 | ⭐⭐⭐ |

> 💡 **一句话总结**：硬核 Agent 技术面，从幻觉治理到 PPO 奖励函数再到多智能体协作，覆盖了 AI 应用开发最前沿的知识体系。
