## 📝 面试问题全记录（10个问题）

### 开场问题
1. **自我介绍**
2. **为什么想找 Agent 应用开发这个工作？**

### 项目深度拷问（旅行Agent项目）

**第一个问题：**  
你说你这个项目是自研的 Harness，那表述一下你的 Harness 都做了什么？

**第二个问题：**  
你的这个 Agent 的分阶段是怎么设计的？是怎么流转的？

**第三个问题：**  
一个简单的例子，你把整个流程描述一遍。

**第四个问题：**  
如果一个用户旅行完之后，会进行下一个旅行，用户在不同旅行当中的各种旅行的经验是如何共享的？你为什么要设计一个外置的状态来进行跟踪，不是完全基于上下文来跟踪状态，你的上下文也没有多长。

**第五个问题：**  
那你的这个记忆系统是怎么设计的？这个记忆的提取和召回分别都是怎么进行的？你的这个记忆提取是怎么实现把各种外部信息归纳为可复用信息的？

**第六个问题：**  
你的这个 Agent 其实依赖各种 API，本质上来说和一个 Web Search 没有什么区别。你认为你这个任务用一个 Web Search 能不能替代？

**第七个问题：**  
你的这个项目，用户凭什么要用你这个项目？为什么不用小红书直接搜一个旅行的结果？

**第八个问题：**  
在我看来，你这个项目其实还是工作流，不是 Agent。当初设计的时候，为什么不用一个全自主性的 Agent？

**第九个问题：**  
全自主性的 Agent 并不是像你说的无法收敛困难。现在先进的全自主性 Agent 在垂类上面也能达到非常强的稳定性。你还是去参考一下现在市面上那个最先进的健康类、医疗类的 Agent 再说吧。

**第十个问题：**  
你有什么想知道的？想问的。

## 💬 作者复盘（4条评论）

### 1️⃣ **回答不好的问题总结** (1小时前) 👍1

**需要回答的几个没有回答好的问题：**

1. **记忆系统的设计**（这一块问得不好，具体细节记不清了）。

2. **有没有关注一些前沿**（关注的有，但是都比较杂，不成体系，答得不够好。我说平时会看 OpenAI 和 Anthropic 的博客，做项目会让 AI 查前沿资讯）。

3. **没有在表达过程中体现 Agentic。**

4. **没有主动从其中一个问题发现面试官的兴趣。**  
具体来说，当面试官认真问我这个项目完全是在一个 session 中进行的时候，就应该感知到他是更喜欢个人助手类的实现。

---

### 2️⃣ **技术细节表达问题** (1小时前)

4. **Session 管理没说清楚。**

5. **强弱 harness 的关系没有说明白。**  
反倒让面试官误解我没有前沿调查。

6. **面试官质疑项目不是 Agent，是 workflow。**  
这块应该解释，但我没说清。

7. **面试官质疑项目用 web search 就能实现，或小红书 search 就能实现。**

8. **小红书 web search 的内容截断问题。**

---

### 3️⃣ **核心技术理念问题** (2小时前)

另一个比较大的问题是，**当面试官质疑这个项目不够 Agentic 的时候，我该怎么回答？**

这一块确实也有点准备不足。我本来是想说去兼顾成本，但是有些面试官可能更看重个人对于技术的前瞻性。

因为我个人其实是能够理解到这一层，就是：
- **当大模型的能力越强，你的 Harness 就需要越轻越薄。**
- **而工作流的这种方法本质上来说在未来是一种负担。**
- 你现在可能用一个轻量的小模型能够实现任务，但未来当模型的成本越来越低的时候，你只需要用一个很轻的边界，就可以实现各种原来精心设计的 Harness 才能完成的任务。

---

### 4️⃣ **总体反思** (2小时前)

一些思考归纳。首先第一点，**对于项目的一些细节掌握的还是不够扎实**，就是没有办法说出：
- 为什么要这么做？
- 其他的方法行不行？
- 能不能有更好的方法？

这一块准备不足，或者说**话术准备不足**。

## 📊 核心问题分类与总结

### 🔍 技术深度考察

**1. 架构设计类**
- Harness 的具体实现和功能
- Agent 分阶段设计与流转机制
- Session 管理机制
- 强弱 Harness 的关系

**2. 状态与记忆系统**
- 外置状态 vs 上下文跟踪的设计选择
- 记忆系统的提取和召回机制
- 跨会话经验共享设计
- 外部信息归纳为可复用信息的实现

**3. 实现细节**
- 完整流程描述能力
- 具体场景举例说明

---

### 💡 产品价值与差异化

**核心质疑：**
1. **与 Web Search 的差异** - 为什么不能用简单搜索替代？
2. **与现有产品的竞争** - 为什么不直接用小红书搜索？
3. **用户价值主张** - 用户凭什么要用你的项目？

**需要准备的回答方向：**
- 深度个性化能力
- 上下文理解和多轮交互
- 知识积累和迁移学习
- 自动化决策和执行能力

---

### 🚀 技术前瞻性考察

**关键质疑：Workflow vs Agent**
- 面试官认为项目更像工作流，不够 Agentic
- 质疑为什么不用全自主性 Agent
- 指出垂类全自主 Agent 已经很稳定（健康类、医疗类）

**需要准备的观点：**
- 工作流与 Agent 的边界和演进路径
- 全自主性 vs 可控性的权衡
- 垂类场景下的稳定性实践
- 成本与能力的平衡考量
- 对未来趋势的理解（模型能力增强 → Harness 轻量化）

---

### 📚 知识储备考察

**前沿追踪：**
- 是否关注 OpenAI、Anthropic 等前沿博客？
- 知识体系是否成体系？
- 是否了解当前最先进的垂类 Agent 实践？

**需要准备：**
- 系统性的前沿知识体系
- 具体的前沿案例和论文
- 对标产品的深度分析

## ⚠️ 面试失败点总结

### 技术表达层面
1. ❌ 项目细节掌握不扎实（为什么这么做？有没有更好的方法？）
2. ❌ 话术准备不足，无法清晰表达技术选型理由
3. ❌ Session 管理等核心概念没说清楚
4. ❌ 强弱 Harness 关系表述不清，导致面试官误解
5. ❌ 记忆系统设计细节记不清

### 产品思维层面
6. ❌ 无法有力回应"为什么不用简单搜索替代"的质疑
7. ❌ 未能清晰阐述项目的差异化价值

### 技术理念层面
8. ❌ "Workflow vs Agent"的本质没有讲清楚
9. ❌ 对全自主性 Agent 的理解有偏差（以为收敛困难）
10. ❌ 对垂类 Agent 的前沿实践了解不足

### 沟通技巧层面
11. ❌ 没有主动捕捉面试官的兴趣点（例如对个人助手类实现的偏好）
12. ❌ 表达过程中没有体现 Agentic 思维
13. ❌ 前沿知识碎片化，不成体系

---

## 💪 改进建议

### 项目准备
1. **对项目每个技术选型都要有充分的理由** - 为什么这么做？其他方案为什么不行？
2. **准备3-5个不同场景的完整流程演示**
3. **总结项目的核心差异化价值** - 用一句话说清楚
4. **准备好应对"为什么不用XX替代"的标准话术**

### 技术深度
5. **梳理清楚 Workflow、Agent、Agentic 的边界与演进**
6. **深入研究垂类全自主 Agent 的前沿实践**（健康、医疗、法律等）
7. **系统学习记忆系统、状态管理的经典设计模式**
8. **理解模型能力增强对 Harness 设计的影响**

### 知识体系
9. **建立系统的前沿知识追踪机制** - 不能碎片化
10. **深度研读 OpenAI、Anthropic、Google 等的技术博客**
11. **关注顶会论文（NeurIPS、ICML、ACL 等）**
12. **研究市面上最先进的 Agent 产品**（Cursor、GitHub Copilot、Claude Projects 等）

### 面试技巧
13. **主动捕捉面试官的兴趣点** - 从提问中识别偏好
14. **表达中要体现 Agentic 思维** - 自主性、适应性、学习能力
15. **准备好"反向提问"** - 了解公司的 Agent 技术栈和方向
