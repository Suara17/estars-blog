## 🎯 你要做到什么级别的小Demo

做一个**在10分钟内用Cursor从零搭一个网页应用**：

```
你直接用Cursor的Composer（或Claude Code）：
1. 输入："帮我用React + TailwindCSS写一个TODO应用，有增删改功能、本地存储、好看"
2. AI生成完整代码
3. 你一键运行，应用跑起来了
4. 你发现一个小bug，高亮代码让AI修复
5. 你加需求："加一个暗黑模式切换按钮"
6. AI秒改

全程10分钟，你写了不到10行代码。
```

**为什么是这个Demo：** 现在JD几乎都写"熟练使用AI编程工具"，面试官可能直接问"你平时怎么用AI写代码"。你光说"会用"太虚了，直接掏出手机给他看你做的TODO应用，然后说"我用Cursor10分钟搞定的"——这就是实证。

## 📄 核心代码框架（不是代码，而是提示词模板）

Vibe Coding的核心不是你写代码，而是你**会写Prompt给AI让它写代码**。

### 场景1：从零生成一个功能（最常用）

```
你 → Cursor / Claude Code：

"帮我用Python + FastAPI写一个RESTful API，功能是管理书籍信息。
需要支持：
- GET /books （列出所有书，支持按作者筛选）
- POST /books （新增一本书）
- GET /books/{id} （查看单本书详情）
- PUT /books/{id} （更新书籍信息）
- DELETE /books/{id} （删除）

数据字段：id、title、author、published_year、isbn
用SQLite存储，用Pydantic做数据校验。
代码要有错误处理、日志记录、清晰的注释。"
```

**关键技巧：** 需求越具体，AI生成的质量越高。不要说"做个API"，要说做什么API、有哪些字段、用什么库、要什么功能。

### 场景2：修改现有代码

```
"选中这段代码 → 按Ctrl+K → 输入：
'这个函数没有错误处理，如果用户传入空字符串会崩溃。
请加上：1. 参数校验 2. try-except 3. 返回合适的HTTP状态码和错误信息'
```

### 场景3：让AI解释代码

```
"选中一段你不懂的代码 → Ctrl+L → 输入：
'逐行解释这段代码在做什么，特别是这个装饰器@router.post的作用，
以及async/await在这里为什么是必要的'"
```

### 场景4：让AI写测试

```
"选中你的函数 → Ctrl+K → 输入：
'用pytest给这个函数写单元测试，覆盖：
1. 正常输入的情况
2. 边界情况（空列表、最大值）
3. 异常输入（None、非法类型）
每个测试用例要有清晰的命名和docstring'"
```

### 场景5：代码审查

```
"选中PR中的变更代码 → Ctrl+L → 输入：
'review这段代码，重点检查：
1. 是否有安全漏洞（SQL注入、XSS等）
2. 性能问题（不必要的循环、重复查询）
3. 代码规范问题
4. 潜在的bug
按严重程度从高到低列出'"
```

## 🛠️ Cursor高效使用指南

```
光标放在某行 → Ctrl+K      → 编辑选中代码（最常用）
光标放在某处 → Ctrl+L      → 对代码提问/解释
Crtl+Shift+L  → 在对话中添加当前文件作为上下文
Ctrl+Enter    → 应用AI的所有修改
Ctrl+Z        → 回退AI的修改（重要！）
@Workspace    → 让AI理解整个项目结构
@Docs         → 让AI查阅官方文档后再回答
@Web          → 让AI联网搜索最新信息
Composer模式  → 多文件编辑
```

## 🧠 Vibe Coding方法论（面试官爱听的）

| 原则 | 怎么说 |
|:----|:------|
| **不要盲目信任** | "我不会直接复制AI的代码，每次都要理解后再提交，经常发现AI生成的代码有逻辑漏洞" |
| **任务要拆小** | "我把一个大需求拆成多个小步骤，每次让AI只完成一小块，逐步集成。一次让AI生成500行代码大概率要重写。" |
| **先写测试再写代码** | "我习惯用TDD模式：先写测试用例，让AI实现能通过测试的代码。这样质量有保障。" |
| **上下文就是一切** | "给AI提供足够的背景信息：项目结构、技术栈、命名规范、编码风格，比一句'帮我写个功能'效果好10倍。" |
| **持续重构** | "AI写的第一版通常可用但不够优雅，我会让它重构：提取公共逻辑、加注释、优化性能。" |

## ❓ 面试官最爱问的3个问题

| 问题 | 满分回答要点 |
|:----|:-----------|
| **1. 你觉得AI编程工具最大的坑是什么？** | ① **幻觉代码**：AI会生成看起来对但实际有问题的代码（如调用不存在的方法）② **上下文丢失**：长对话中AI会忘记之前的约定 ③ **过度工程**：AI倾向于生成过度复杂的代码 ④ **安全风险**：AI可能引入不安全的代码（如SQL拼接）。**解法：永远先review再提交。** |
| **2. 你怎么确保AI生成代码的质量？** | ① 用TDD约束：先写测试再让AI生成 ② **分步骤**：一次只改一个功能，改完立即测试 ③ **Code Review**：逐行检查AI代码 ④ **让AI自己review自己的代码**：生成完后让AI检查潜在问题 ⑤ **版本控制**：AI的修改用Git管理，方便回退。 |
| **3. 你一天写多少行代码？用AI之后效率提升了多少？** | 之前一天写200-300行有效代码，现在能写到800-1200行，**效率提升3-4倍**。但更重要的是质量：以前写CRUD占用了大量时间，现在可以让AI处理样板代码，**我把精力集中在架构设计、业务逻辑和测试上**。 |
