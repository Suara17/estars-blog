## 🎯 从这份面经可以吸取的教训

### 项目准备方面
1. **项目来源要会包装** — 不要直接说"网上找的"
2. **项目挑战要有完整叙事** — 问题 → 方案设计 → 量化收益
3. **Agent 相关的高频深度题**必须准备：
   - 短期/长期记忆实现方案
   - Agent 架构设计（至少从规划→工具→记忆→执行→反馈闭环说清楚）
   - 幻觉处理层级（提示词约束 → 工具/函数调用 → RAG 召回 → 微调对齐）
   - API 超时、报错、token 消耗的排查和优化策略（缓存、批处理、上下文压缩、降级）
   - 结构化提示词设计原则

### 八股基础方面
4. **不能只会背"是什么"** — 面试官会追问"你怎么设计一个 X"（如线程池、Agent架构）
5. **多线程基础不能丢** — 死锁、锁、并发编程是高频考点
6. **JVM 类加载机制** — 经典必问题
7. **MySQL 日志体系** — undo log、redo log、binlog + 两阶段提交，属于必考八股组合
8. **刷过的算法题，面试时假装思考一下再写**
