---# 小万工作间 · Karpathy 编译体系变更说明与维护指南

> **创建日期：** 2026-06-19
> **变更依据：** Karpathy 知识库方法论（raw → wiki → outputs 三层编译架构）
> **执行人：** 小万 AI

---

## 一、为什么改

小万工作间的目录结构原本是按"内容类型"分类的（外部精选、求职作战室、研究实验室等），角色清晰，但存在几个问题：

1. **raw 和 wiki 混在一起** — 同一个目录下既有原始素材（面经原文）也有编译产物（知识点、合集），分不清"原材料"和"成品"
2. **没有索引层** — 跨目录的知识（比如小红书里也有面经）只能靠人脑记忆
3. **对话蒸发了** — 每次和 AI 讨论的复杂结果留在聊天记录里，没有沉淀成文件
4. **没有健康检查** — 没有机制检查笔记的一致性、完整性

这套改动引入 Karpathy 的"编译"概念：**Obsidian Vault 是代码仓库，LLM 是编译器**。

## 二、改了什么

### 2.1 求职作战室 — 面经 raw/wiki 分离

**变动内容：**
- `求职作战室/面经/` → 回归 **raw** 角色，只放 `[面经]` 原始帖子
- `求职作战室/知识点提炼/` → **新建**，存放 `[知识点]` 类编译产物（从面经/移入 26 篇）
- `求职作战室/合集笔记/` → **新建**，存放 `[合集]` `[笔记]` `[项目]` 类编译产物（从面经/移入 9 篇）
- `求职作战室/索引/面经来源索引.md` → **新建**，跨目录追踪面经原始来源
- `求职作战室/面经/cleanup.sh`、`.backup/`、残留 `.custom_clean` 文件 → **已清理**

**影响：**
- `面经/` 目录从 ~60 项减少到 ~34 项（仅保留标记为 `[面经]` 的文件）
- 知识点和合集笔记现在有了自己独立的空间

### 2.2 外部精选 — 新增 wiki 编译层

**变动内容：**
- `外部精选/wiki/indexes/` → **新建**（来源索引，后续逐步填充）
- `外部精选/wiki/concepts/` → **新建**（概念提炼，后续逐步填充）
- `外部精选/wiki/summaries/` → **新建**（结构化摘要，后续逐步填充）
- `外部精选/.backup/` → **已清理**

**说明：** 各平台子目录（公众号精选、小红书精选、L站精选、外文翻译）**物理不动**，新增的 wiki 层用于存放 AI 编译产物。跨目录的面经素材通过 `求职作战室/索引/面经来源索引.md` 链接。

### 2.3 研究实验室 — 新增 raw 层

**变动内容：**
- `研究实验室/raw/PDF/` → **新建**（存放原始论文 PDF）
- `研究实验室/raw/原文摘录/` → **新建**（存放论文关键段落原始引用）
- `研究实验室/论文与调研/` → **保持不动**（wiki 层）

### 2.4 工程现场 — 可选 raw 层

**变动内容：**
- `工程现场/raw/工作日志/` → **新建**（可存放调试日志、踩坑截图等原始素材）

**说明：** 如果你平时不习惯留原始日志，这个目录可以暂时不用。

### 2.5 新增全局层

**变动内容：**
- `wiki/indexes/全库来源索引.md` → **新建**（所有 raw 来源的总清单）
- `wiki/indexes/全库概念索引.md` → **新建**（跨目录核心概念索引）
- `outputs/qa/` → **新建**（复杂问答沉淀目录）
- `outputs/health/` → **新建**（健康检查报告目录）

### 2.6 其他

- `待整理/临时收纳/` → **新建**（不确定放哪的文件暂存区）
- 代码/项目目录（estars-blog、my_openclaw、vibe-coding-cn-develop）→ **不动**
- 素材目录（博客素材包）→ **不动**
- 个人日记（回音壁）→ **不动**
- 纯 wiki 目录（航海图/操作指南/、工程现场/实战.经验/）→ **不动**

## 三、新工作流

### 日常流程

```
你从外部获取内容
     │
     ▼
┌─────────────┐
│  raw/       │  ← 原始素材，只存不改
│  (各目录)   │       你说"存个面经" 或 "存篇公众号"
└──────┬──────┘
       │ "帮我编译一下新增的 raw"
       ▼
┌─────────────┐   ← 小万作为编译器
│  编译器     │
│  (小万 AI)  │
└──────┬──────┘
       │
       ├──→ wiki/summaries/  ← 结构化摘要（增量产出）
       ├──→ wiki/concepts/   ← 核心概念提炼（增量产出）
       ├──→ wiki/indexes/    ← 来源索引更新
       ├──→ outputs/qa/      ← 复杂问答落文件
       └──→ outputs/health/  ← 每周健康检查
```

### 你的 4 种常用指令

| 你说 | 我做 |
|------|------|
| "存个面经/公众号/小红书" | 存到对应 raw 目录 |
| "编译一下新增的 XX" | 读新增 raw，产摘要、提概念、更新索引 |
| "更新一下 XXX 操作指南" | 读相关 raw/wiki，产出更新 |
| "做一次健康检查" | 扫 wiki 层，找概念冲突、缺定义、孤岛笔记 |

## 四、维护说明

### 每周维护清单（可选）

1. **清理待整理/** — 把暂存区的文件分类到对应目录
2. **编译新增 raw** — 告诉我"编译一下最近新增的 raw"
3. **健康检查** — 告诉我"做一次健康检查"
4. **清理外部精选/.backup/ 等临时文件**
