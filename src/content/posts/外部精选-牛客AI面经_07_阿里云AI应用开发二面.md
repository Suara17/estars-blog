# ☁️ 阿里云 AI 应用开发二面 面经

> **作者**：工作小梦想  
> **发布**：2025-05-14  
> **来源**：牛客网（搜索结果页摘要，全文因登录限制不可访问）  
> **备注**：本题量较少（8题），但方向纯正，与已有阿里一面（Java Agent方向）形成互补

## 📋 面试题目

### 项目深挖与技术选型

1. 自我介绍
2. **项目深挖**：详细讲解你做过的 AI 相关项目，你在其中承担的角色？
3. **Multi-agent 工作流设计**：你的项目涉及到多个 Agent 协作，能介绍一下整体的工作流设计吗？为什么选择这种 Multi-agent 架构？
4. **选型理由**：相比直接调 API 或单 Agent 方案，Multi-agent 架构带来了什么收益？又引入了哪些新挑战？

### 知识库与检索优化

5. **知识库搭建**：项目中的知识库是如何构建的？用的是什么向量数据库？分块策略是怎样的？
6. **知识库性能优化**：当知识库文档量大时，如何保证检索的实时性和准确性？有没有做什么优化手段？

### 业务落地与工程能力

7. **AI 在项目中的占比**：你的项目里 AI 相关的工作占多少比例？非 AI 的部分（工程化、运维等）你是如何处理的？
8. **AlCoding 日志分析系统**：介绍一下你实现的 AlCoding 日志分析系统？它解决了什么痛点？整体架构是怎样的？

## 📊 知识点图谱

| 领域 | 知识点 | 重要度 |
|:---:|---|:---:|
| Multi-agent | 架构设计 / 选型对比 / 协作模式 | ⭐⭐⭐⭐⭐ |
| 知识库 | 向量数据库 / 分块策略 / 检索优化 | ⭐⭐⭐⭐ |
| 工程能力 | 日志分析 / AlCoding / 全栈思维 | ⭐⭐⭐ |
| 项目表达 | 角色定位 / 难点突破 / 量化收益 | ⭐⭐⭐⭐ |

> 💡 **一句话总结**：Multi-agent 架构设计与知识库工程双核驱动，侧重项目深挖与选型理由，适合补全 Agent 落地的工程化视角。
